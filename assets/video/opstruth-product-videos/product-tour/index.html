product tour

A practical OpsTruth run, from first command to evidence.

Everything shown here is grounded in the current CLI surface.

install
$ npm install -g opstruth $ opstruth welcome
Install globally, or run it one-off with npx.
welcome
Welcome to opstruth. This tool runs read-only operational checks to help you understand: - what changed - what is configured - what looks risky - what was verified - what was not verified It will not deploy, mutate databases, trigger jobs, publish content, restart services, call AI providers, or print raw secrets.
The first screen explains the safety boundary before any proof is collected.
optional config

Add explicit inputs when route or runtime proof matters.

$ opstruth init --yes Created opstruth.config.json routes: ["/", "/health"] local: ports + health paths secrets: allowlist paths when reviewed
Config improves confidence, but the command still only checks the inputs you provide.

secret scan

Risky references are evidence, not exposed values.

$ opstruth secrets warning: risky secret reference file: src/config.ts line: 12 preview: ***REDACTED*** next: move real values to secret storage
Secret scans show where to look while keeping raw values out of terminal output.
runtime inputs

Route and local checks are explicit-input probes.

$ opstruth routes --base-url https://example.com HEAD / 200 GET /health 200 headers checked production still means observed-at-this-time
$ opstruth local --port 3000 --health /health port listening health path responded service not started process not killed
OpsTruth never starts or repairs services just to make a check look green.
hosted quality proof

CI evidence only counts when it matches the exact commit.

$ opstruth github-ci --workflow CI exact commit match: yes workflow: CI conclusion: success CI proof remains separate from production proof.
A successful workflow for another SHA is reported as a proof gap.
production evidence files

Supabase live proof is explicit and offline.

$ opstruth supabase-live --evidence-file redacted.json verified: function deployed verified: scheduler configured verified: autonomous run observed not verified: filtered function logs unsafe to test: rate-limit branch
It validates reviewed redacted evidence without querying production or inspecting secrets.
handoff
$ opstruth --json --skip evidence machine-readable result ANSI-free output verified, warnings, failures, skipped, not verified next safe step included
