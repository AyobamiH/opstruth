# opstruth v0.2 Backlog

## Real-Repo Evidence

- Add better real-repo case study support.
- Add evidence summaries that distinguish source review, PR review, local checks, and live runtime checks.
- Add first-class "what was deliberately not done" output.
- Add a first-class "pre-application validation" state for work that is ready but not approved for mutation.
- Add readiness-only evidence sections for reviewed execution packets that remain unexecuted because approval is absent.
- Make configured local route/runtime failures distinguish between a failed runtime and a missing required local preview precondition.

## Supabase And Scheduler Proof Gaps

- Make Supabase static versus live verification clearer in CLI output.
- Improve scheduler and cron proof-gap language.
- Separate guarded migration drafts from applied production changes.
- Make remote secret setup and scheduler mutation gates explicit.
- Represent guarded non-executable scheduler migrations as planned application evidence, not applied production proof.
- Add first-class live Supabase evidence sections that separate remote secret-name setup, function deployment, denial paths, authorised smoke requests, scheduler configuration, scheduler execution, telemetry, and database effects.
- Add and harden `opstruth supabase-live` as an evidence-file-driven command for redacted production Supabase proof.
- Preserve a distinct `scheduler configured` versus `scheduler executed` state.
- Add count-only database-effect summaries for approved production smoke checks.

## Secret-Boundary Evidence

- Keep improving secret-boundary evidence summaries without exposing values.
- Keep identifier-name matches distinct from secret-value findings.
- Extend grouped secret-reference findings into evidence pack summaries and case-study views.
- Keep redaction behavior conservative by default.
- Preserve regression coverage for documentation references, placeholders, local-only files, unknown token-like strings, and actionable source findings.

## GitHub Handoff Evidence

- Explore stronger GitHub PR/evidence handoff integration.
- Summarize intended files, checks, PR state, and merge state in evidence packs.
- Keep deployment, database mutation, release, and production verification as explicit non-default gates.

## Quality Failure Summaries

- Preserve clean-lint evidence separately from runtime and production proof gaps.
- Show the failed script name, exit code, and concise next action when local quality gates fail.
- Preserve `STATUS: Fail` for failed quality checks even when repo inspection and JSON parsing succeed.
- Keep quality proof signals distinct for lint, typecheck, tests, build, and CI.
- Extend first-class GitHub Actions ingestion beyond `gh` CLI metadata, if needed, while preserving exact-commit matching.
- Keep GitHub CI proof separate from local quality proof and production runtime proof in evidence packs.
- Add CI-aware case-study output that separates local quality proof, GitHub-hosted proof, and production runtime gaps.
- Preserve regression coverage for commit SHA redaction boundaries in exact-commit CI proof.
- Improve presentation of `github-ci` results in one-command summaries and evidence packs.
- Make missing route/local-runtime inputs actionable without implying runtime health.
- Detect multiple lockfiles and warn when the selected package manager is unavailable.
- Track before/after quality counts during repeated real-repo validation runs.
- Keep regression coverage for one-command local orchestration honoring local runtime inputs declared in `opstruth.config.json`.
- Summarize which configured route and local subchecks actually ran in one-command output.
- Keep regression coverage that distinguishes local-preview header warnings from deployed production header evidence.
- Consider richer environment labels for staging and preview URLs that are not loopback hosts.
