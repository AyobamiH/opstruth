## Package Checks

These checks passed from `cli/`:

```text
npm run lint
npm test
node bin/opstruth.js --help
node bin/opstruth.js welcome
node bin/opstruth.js probes
node bin/opstruth.js --color --skip evidence
node bin/opstruth.js --no-color --skip evidence
NO_COLOR=1 node bin/opstruth.js --skip evidence
node bin/opstruth.js --json --skip evidence
npm pack --dry-run
```

The JSON output parsed successfully and was ANSI-free. In this sandbox, the JSON run needed `NPM_CONFIG_CACHE=/tmp/opstruth-npm-cache` plus filesystem escalation so the build probe could write ignored build output. The final JSON result had no failures.

`npm pack --dry-run` reported:

```text

```

## npm Publish

`opstruth@0.1.2` was published to npm as public.

The first publish attempt reached npm packaging but failed with registry DNS `EAI_AGAIN`. The retry with network escalation succeeded.

npm emitted an auto-correction warning for the `bin` path during publish. The published package metadata was verified afterward:

```json
{
  "version": "0.1.2",
  "bin": {
    "opstruth": "bin/opstruth.js"
  },
  "dist.tarball": "https://registry.npmjs.org/opstruth/-/opstruth-0.1.2.tgz"
}
```
